## 1. 產品定位

### 一句話定義
KOVA 是一個 AI 驅動的美股資訊觀察平台，幫助一般投資人整理市場資訊，做出更有根據的自主決策。

> **瀏覽器頁籤標題**：`KOVA AI - 美股資訊觀察平台`（不使用「投資分析」等受《證券投資顧問事業管理規則》規範之用語）。

### 目標用戶
| 用戶類型 | 描述 |
|---------|------|
| 散戶投資人 | 有投資美股但缺乏系統分析能力的個人 |
| 理財愛好者 | 對股市有興趣，想要 AI 輔助整理資訊 |
| 台灣用戶 | 繁體中文介面為主，也支援英文 |

### 核心價值主張
- 不需要懂技術分析，AI 幫你整理關鍵訊號
- **18 維度整合分析**：12 項量化訊號 × 6 種投資風格觀點
- 即時美股股價 + AI 深度資訊整理合而為一

> **重要定位**：KOVA 不是投資顧問服務，所有輸出皆為 AI 生成的市場資訊整理，不構成投資建議。使用者必須自行做最終決策。

## 3. 用戶旅程

### 未登入用戶
1. 進入首頁 → 看到 Hero（「洞見市場，由你掌舵」＋ 美股 AI 分析 badge）
2. 點「市場分析」或「個股預測」→ 直接進入該功能頁（不需先「開始試用」按鈕）
3. 選擇類型與時間 → 點「建立 AI 推薦任務」
4. 首次分析自動啟動試用（`trial_state.active = true`，計 1 次）
5. 等待 30~40 秒 → 看到 3 支推薦股 / 預測結果
6. 用完 5 次 → 下次點分析跳出「請登入帳號」Modal

### 登入用戶
1. 登入後 → 無使用次數限制（初期開放）
2. 分析結果自動存入歷史紀錄（localStorage）
3. 可將股票加入自選追蹤

## 5. 產品邊界（不做什麼）

- ❌ 不提供台股（僅美股）
- ❌ 不執行實際交易
- ❌ 不保證投資報酬
- ❌ 所有分析僅供參考，不構成投資建議（法律免責）
- ❌ 不發放或使用「投資顧問」「財務顧問」「投資分析」等受管制用語
- ❌ 不回答「我該不該買？」「會不會漲？」等個人化投資指示

## 6. 法律合規機制（Legal Compliance Layer）

> 目的：在台灣《證券投資信託及顧問法》§4／§7 架構下，讓 KOVA 明確定位為「AI 資訊整理工具」，而非未具牌照的投資顧問服務。

### 6.1 縱深防禦（Defense in Depth）
| 層級 | 機制 | 位置 |
|------|------|------|
| L1 | 首訪強制同意牆（三項勾選） | `LegalConsentModal`，覆蓋整個 App |
| L2 | 註冊時二次勾選（年滿 20 + 自負盈虧） | `LoginModal` 註冊模式 |
| L3 | 結果頁警語帶（紅底 AlertTriangle） | 三種分析結果頂端 |
| L4 | Footer 免責聲明 + ToS / Privacy 連結 | 全站可見 |
| L5 | 術語軟化（預計目標→觀察目標、停損價位→風險觀察線等） | 全站 i18n 文案 |
| L6 | 瀏覽器標題避開「投資分析」受規範用語 | `index.html` `<title>` |

### 6.2 術語轉換對照
| 舊用語（受管制風險高） | 新用語 |
|---------------------|-------|
| 投資建議 / Advice | 市場分析 / Market Analysis |
| 推薦股票 | 研究標的 / Research Idea |
| 信心指數 / Confidence | 訊號強度 / Signal Strength |
| 準確度 / Accuracy | 訊號符合度 / Signal Match |
| 買入 / 賣出訊號 | 價格參考點 / 保護性停損 |
| 預計目標（個股預測） | 觀察目標 |
| 停損價位（個股預測） | 風險觀察線 |
| 目標價位線 / 停損價位線（圖表） | 觀察目標線 / 風險觀察線 |
| 智慧投資分析平台（頁籤標題） | 美股資訊觀察平台 |

### 6.3 同意狀態儲存
- `localStorage.kova_legal_consent = "1"`（首訪同意後）
- `localStorage.kova_legal_consent_at = <ISO timestamp>`（稽核用）
- 使用者清除 cache 或換裝置會再次觸發同意牆

### 6.4 維運責任
- ToS / Privacy 文字由產品團隊維護於 `src/i18n.ts`，上線前須律師審閱
- ToS 含 KOVA 營運者識別（`legal.terms.operator.*`），Privacy 含聯絡信箱（`legal.privacy.contact.*`）
- 術語若再調整，須同步更新中英文 i18n key
- 任何新增的分析模組必須套用 `<ResultDisclaimerBanner />`
